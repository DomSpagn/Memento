Memento — Release Notes

Full history of features and fixes

v1.8 April 2026
✦ New Features
  • Archive Path: the Installation Path concept has been redesigned. The wizard step 3 now asks for a parent folder; Memento automatically creates a Memento/ subfolder inside it (with TaskTracker/ and DesignTracker/ sub-directories). The resulting …/Memento path is saved as OutputPath in mem_conf.json.
  • Archive Path in Settings: the Settings → Installation Path… menu item has been renamed to Archive Path…. The dialog now requires the selected path to end with a folder named Memento (validation popup shown otherwise). Saving updates OutputPath only — no new folders are created — and automatically refreshes both tracker tables.
  • Status field locked when history exists: the Status dropdown in the Task and Design header becomes read-only as soon as the record has at least one history entry. Status is then derived exclusively from the history entry statuses via automatic computation.
✦ Bug Fixes
  • Status not updated when adding an Open history entry: adding a new history entry with status Open (the default) did not trigger automatic parent-status recomputation. The status is now always recomputed immediately after saving any new entry, regardless of its status value.
v1.7 April 2026
✦ Bug Fixes
  • PDF report — attachment icon in history entries: the attachment indicator shown next to file names inside history update entries was rendered as an empty square (□) because the 📎 emoji falls outside the WinAnsi character range supported by standard PDF fonts. It is now replaced with a proper envelope glyph (✉) from the built-in ZapfDingbats font, which renders correctly in all PDF viewers. Each attachment is also listed on its own line instead of being joined by commas.
v1.6 April 2026
❆ Improvements
  • Status chart: period filter removed: the Period dropdown (Last Day / Last Week / Last Month / Last Year) has been removed from the Status Distribution chart in both Task Tracker and Design Tracker. The chart now always shows the distribution across all records, filtered only by project.
  • Revised automatic parent-status computation: the rule set that derives a task/design status from its history entries has been updated. Open now wins over Closed whenever at least one entry is Open (previously a mix of Open + Closed left the status unchanged). Updated priority rules:
    1. At least one In Progress → parent becomes In Progress.
    2. At least one On Hold → parent becomes On Hold.
    3. At least one Open → parent becomes Open.
    4. All Closed → parent becomes Closed.
v1.5 April 2026
✦ New Features
  • Per-entry status in History: each history entry card now has a compact status dropdown in the bottom-right corner. Available values: Open, In Progress, On Hold, Closed. The selected status is shown with a colour-coded text label matching the palette of the main table chips.
  • Status dropdown on new update form: when composing a new history entry the status dropdown appears immediately next to the file-attach button, defaulting to Open. The selected status is saved together with the entry.
  • Automatic parent-status computation: after any history entry status is set or an entry is deleted, the parent task / design status is recomputed automatically according to the following priority rules:
    1. If any entry is In Progress → parent becomes In Progress.
    2. Else if any entry is On Hold → parent becomes On Hold.
    3. Else if all entries are Closed → parent becomes Closed.
    4. Else if all entries are Open → parent becomes Open.
    5. Mixed states (some Open, some Closed) → parent status is left unchanged.
  • Σ combined Math & Equations button: the previous separate f(x) (Math symbols) button has been replaced by a single Σ button that opens a two-tab dialog:
    • Math symbols tab — same symbol set as before (±∞≠~×÷∝≪≫≤≥∓≅≈≡∂√∪∩∅°∆∇∃∄∈∋←↑→↓↔∙⋯⟸⟹⟺).
    • Equations tab — ready-made equation templates grouped by category: Algebra, Calculus, Trigonometry, Notable formulas.
    The button is present in all six toolbars (Description + new entry + existing entry, in both trackers).
✦ Bug Fixes
  • Cursor focus after toolbar actions: clicking any formatting button (Bold, Italic, Underline, bullet list, indent, Greek / Math / Equations pickers, etc.) no longer moves input focus away from the text field. The cursor remains on the active line and the insertion point is preserved.
✦ Improvements
  • Tag deduplication: adding a hashtag that already exists in the same history entry is silently ignored — no duplicate tags are appended.
  • Tag same-line append: when a tag is added to an already-saved history entry that ends with a tag-only line, the new tag is appended on the same line instead of on a new line.
  • History database schema extended: the history table gains an entry_status TEXT NOT NULL DEFAULT 'Open' column. Existing databases are migrated transparently on first run.
v1.4 April 2026
✦ New Features
  • Special character pickers: two new buttons added to the formatting toolbar of Description and History fields in both Task Tracker and Design Tracker.
    • Ω — opens a dialog with the full Greek alphabet (uppercase and lowercase). Click any character to insert it at the cursor position.
    • f(x) — opens a dialog with a set of mathematical symbols: ±∞≠~×÷∝≪≫≤≥∓≅≈≡∂√∪∩∅°∆∇∃∄∈∋←↑→↓↔∙⋯⟸⟹⟺. Click any symbol to insert it at the cursor position.
  • History entry creation date: each history entry card now shows both the Created date and a Modified date (only displayed if the entry has been edited after creation).
✦ Improvements
  • History database schema extended: the history table gains a modified_at column, populated automatically when an entry is edited. Existing databases are migrated transparently on first run.
v1.3 March 2026
✦ New Features
  • Cancel button (✕) added to each existing history entry card. The button appears when edit mode is active and restores the original text if clicked — no changes are saved to the database.
  • Duplicate attachment prevention: attempting to attach a file whose name already exists in the attachments folder now shows a warning dialog that lists every Task and Design the file is already linked to (including history entries), instead of silently overwriting or duplicating it.
✦ Bug Fixes
  • History entry formatting toolbar and tag section now appear immediately when clicking the Edit button on a history card, instead of requiring at least one character to be typed first.
  • Deleting a task or design via the toolbar button now also removes all associated attachment files from disk (previously only the database record was deleted, leaving orphaned files in the attachments folder).
✦ Improvements
  • All header fields in the Task and Design detail pages are now auto-saved on every change — no Save button is required. Affects: Title, Project, Status, Alarm (Task); Name, Board, Revision, Project, Category, Function, Status (Design).
  • Attachment file naming convention changed: files are now saved as {original_name}_Task_{id}.{ext} for task attachments and {original_name}_Design_{id}.{ext} for design attachments, making it easy to identify which record each file belongs to.
  • Installer updated to v1.3: Memento_Setup_1.3.exe produced by build.bat via PyInstaller + Inno Setup. Fixed invalid WizardStyle directive that prevented installer compilation.
v1.2 March 2026
✦ Improvements
  • Attaching a file in the Files section now immediately enables the main Save button (Task Tracker & Design Tracker).
  • Attaching a file to an existing history entry now enables the main Save button.
  • Save update now works even when only file attachments are added to a new history entry — a text body is no longer required.
  • History entry cards simplified: the bottom bar (per-entry Save and Cancel buttons) has been removed. The Edit button now toggles to a ✓ (Save) icon when in edit mode; clicking it again saves the entry. The Attach (📎) button is always visible in the card header.
  • Added a red 🗑 Delete entry icon button to each history card header. Deleting an entry also removes all its file attachments from disk.
  • The main action bar (Delete / Cancel / Save buttons at the bottom of the task or design detail page) has been removed. Changes are saved automatically when navigating back; a red 🗑 icon in the form header allows deleting the record.
  • Staged file attachments for a new history entry are now shown as a list below the text field, with individual remove (×) buttons, giving clear visual feedback before saving.
v1.1 March 2026
✦ New Features
  • History updates are now added via a dedicated + New update button instead of an always-visible text field.
  • Formatting toolbar (Bold, Italic, Underline, Bullet list, Numbered list, Text colour, Quote, Remove quote) is shown immediately when composing or editing a history entry.
  • Tag section and Attach button appear directly after clicking + New update, without needing to type first.
  • Each existing history entry now has its own formatting toolbar, tag section and Attach button, shown when the entry is put in edit mode.
  • Attachments added while composing a new history entry are saved atomically together with the entry.
  • Release Notes page (this file) accessible from the Help menu.
✦ Bug Fixes
  • Fixed crash ("str object is not callable") when adding a tag via the + button — loop variable shadowed the translation function.
  • DataTable now stretches to fill the full panel width in both Task Tracker and Design Tracker.
  • System tray icon rendered correctly on Windows 11 (DPI awareness + RGBA conversion).
  • Taskbar pin now relaunches Memento instead of a blank flet window.
