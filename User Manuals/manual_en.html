Memento

User Manual  ·  English

Memento is a desktop application for engineering teams that need to manage work items and technical documents in one place. It provides two dedicated modules: the Task Tracker for action items and the Design Tracker for hardware or software design artefacts (schematics, PCB layouts, block diagrams, etc.).

All data is stored locally in SQLite databases inside a folder of your choosing. Attachments are copied to that folder automatically. The application also runs a background system-tray process that delivers Windows toast notifications for task alarms even when the main window is closed.

1 · Getting Started

1.1 System Requirements & Installation

ComponentRequirement
Operating SystemWindows 10 or Windows 11 (64-bit)
DisplayMinimum 1280 × 768 recommended

Installing from the setup wizard (recommended)

Run Memento_Setup_1.3.exe and follow the on-screen instructions. The installer copies Memento.exe, MementoTray.exe, and all required files to a folder of your choice (default: %ProgramFiles%\Memento) and optionally creates a desktop shortcut. No Python installation is required.

Running from source (developers)

ComponentRequirement
Python3.10 or later
Key librariesflet, pystray, Pillow

Install dependencies with:

pip install -r requirements.txt

1.2 First-Run Setup Wizard

The first time you launch Memento (by running python main.py), the Setup Wizard opens automatically because no configuration file exists yet. It walks you through four steps:

Step 1 of 4 — Theme

Choose between Dark Mode (default) and Light Mode. The application immediately applies the selected theme so you can preview it before confirming. Your choice is saved to the configuration file.

Step 2 of 4 — Output Folder

Choose the root folder where Memento will store all its data (databases, attachments, etc.). A Memento/ sub-folder is created automatically under the path you choose. The default value is ~/Documents. Click the folder icon button to open a system directory picker.

Step 3 of 4 — Starting Application

Select which tracker should be shown when Memento starts: Task Tracker or Design Tracker. Click Finish ✓ to complete the wizard; the main window opens immediately.

Step 4 of 4 — Auto-start

Enable or disable automatic startup of Memento's system-tray process at Windows boot. Toggle the on/off switch to your preference. If enabled, the command python tray_app.py --install is run immediately in the background; if disabled, no action is taken during first-run setup.

ℹ️  You can change all four of these preferences later via the Settings menu.

1.3 Output Folder Structure

After the wizard completes, Memento creates the following structure on disk:

{InstallationPath}/
└── Memento/
    ├── TaskTracker/
    │   ├── db/
    │   │   └── tasks.db         ← SQLite database
    │   └── attachments/         ← Task file attachments
    └── DesignTracker/
        ├── db/
        │   └── designs.db       ← SQLite database
        └── attachments/         ← Design file attachments

2 · Main Window

The main window consists of a top app bar and a central content area that switches between the Task Tracker and Design Tracker views.

2.1 Tracker Switcher

The left side of the app bar contains a segmented button with two segments:

  • Task Tracker — Deep orange thumb
  • Design Tracker — Blue thumb

Click either segment to switch trackers. The last selection is saved to the configuration file (StartWith key) and restored on the next launch.

2.3 Settings Menu

Click the ⚙ icon on the right side of the app bar to open the Settings pop-up menu. It contains four items:

  • Installation Path… — Change the root folder where Memento stores data. Contains a text field showing the current path and a Browse… folder button. Buttons: Cancel  |  Save.
  • Command Bar… — Choose where the toolbar appears (Top / Bottom / Left / Right). Buttons: Cancel  |  Apply.
  • Language… — Switch the interface language between English and Italian. Buttons: Cancel  |  Apply. Applying the change rebuilds the entire UI immediately.
  • Auto-start… — Enable or disable automatic startup of the system-tray process at Windows boot. An on/off toggle switch is shown in the pop-up. Buttons: Cancel  |  Apply.

2.4 Theme Toggle & Help Menu

  • Theme toggle (🌙 / ☀️ icon) — Switches between Dark and Light themes in real time. The preference is saved immediately to the config file.
  • Help menu (? icon) — Contains two items:
    • About — Shows the application version, build date, and author.
    • User Manual — Opens this manual.

3 · Task Tracker

The Task Tracker manages work items. Each task has a title, project, status, optional alarm, description, related items, file attachments, and a history log.

3.1 Task Table

Tasks are displayed in a sortable table with the following columns:

ColumnSortableNotes
#—Auto-incremented integer ID
Title—Clickable orange link — opens the task detail view
Project↕Free text
Opened↕ISO datetime, grey text
Modified↕ISO datetime, grey text
Closed↕Populated when status is set to Closed
Status↕Colour chip (see below)
Alarm—🔔 green = active alarm; 🔕 red = no alarm or already fired

Status colours:

Open In Progress On Hold Closed

Click a row to select it (blue highlight) and enable the Edit and Delete toolbar buttons. Click the same row again to deselect. Click the orange title link to open the task without selecting it.

💡  Column headers with ↕ arrows are clickable — click once to sort ascending, again for descending, a third time to reset to default.

3.2 Creating a New Task

Click the New Task button (or the green + icon) to open the New Task dialog.

Required fields

  • Title — The task name (auto-focused on open).
  • Project — The project this task belongs to. As you type, a live autocomplete dropdown shows up to 6 suggestions from existing projects. Click a suggestion to fill the field.
  • Status — Dropdown: Open In Progress On Hold Closed. Default: Open.

The Save button is disabled until all three required fields are filled in.

Optional fields in the New Task dialog

  • Alarm — See § 3.4 Alarm Section.
  • Description — Multiline free text (3–5 lines).
  • Related Tasks — Link existing tasks by ID.
  • Related Designs — Link existing designs by ID.
  • Files — Attach one or more files.

3.3 Editing a Task

Open a task by clicking its title link in the table or by selecting its row and pressing the Edit Task toolbar button. The full-page detail view opens, showing a breadcrumb "← Back" arrow in the app bar and the task number as the page title.

All header fields are immediately editable:

FieldType
TitleInline text field (full width, underline style)
ProjectInline text field + live autocomplete dropdown
StatusDropdown (Open / In Progress / On Hold / Closed)

Every change is saved automatically to the database as you type or select — no Save button is required. To permanently remove the task, click the red 🗑 icon button in the top-right corner of the header section.

3.4 Alarm Section

3.6 Related Items & File Attachments

Related Tasks

In the Related Tasks section, type a task ID number in the input field and click the green ✓ button (or press Enter) to create a bi-directional link between the current task and the referenced task. Each linked task is shown as: #ID   Task Title with a 🔗 remove button.

Inline error messages:

  • Enter a valid number — non-numeric input
  • Already added — the ID is already linked
  • Task not found — no task with that ID exists

Related Designs

Same behaviour as Related Tasks, but links to designs instead. Enter a design ID.

Files

Click Attach File (📎) to open a file picker. The chosen file is copied to {InstallationPath}/Memento/TaskTracker/attachments/ and renamed to {original_name}_Task_{id}.{ext} so it is always traceable to the task that owns it. Each attachment is listed with a clickable filename (opens with the system default application) and a delete button (removes the attachment from the list and from disk). Attaching a file is saved immediately — no further action is required.

If the selected file is already attached elsewhere (in another Task, Design, or any history entry), a warning dialog is shown listing all the records that already contain that file. The attachment is not added in that case.

3.7 History Log

The History section at the bottom of the task detail view provides a chronological log of updates. It is useful for tracking progress notes, decisions, and status changes over time.

Adding a new entry

  1. Type your update in the Write an update… text field (blue-tinted, 2–4 lines).
  2. Optionally add hashtags: enter a tag in the #tag field and click the + button. Tag chips appear below the field; click × to remove one.
  3. Optionally click the 📎 attach button to add one or more files to the entry.
  4. Click Save update (blue button). Tags are appended to the entry body automatically. You can save with just file attachments even if no text has been typed.

Existing entries

Each historical entry is shown as a card with:

  • Header: Created: relative date + Modified: relative date (only shown if the entry was edited after creation) + entry number + 📎 Attach + 🗑 Delete + Cancel (✕, visible during editing) + Edit / ✓ Save toggle button.
  • Body: the update text (read-only by default). Click Edit to enter edit mode — the formatting toolbar and tag section appear immediately. Click ✓ to save. Click ✕ Cancel to discard all changes and restore the original text.
  • Attached files: files linked to that specific entry, with the same attach/open/delete controls as the main Files section.

3.8 Deleting a Task

Click the red 🗑 icon button in the top-right corner of the task header, or select a row in the table and press the Delete Task toolbar button. A confirmation dialog appears: "Are you sure you want to permanently delete this task?"

⚠️  Deletion is permanent. The task record, all history entries, and all attachment files on disk are removed. This cannot be undone.

3.9 Filtering & Searching Tasks

Filter

Click the Filter toolbar button to open the Filter Tasks dialog. Use the dropdowns to narrow the list:

  • Project — distinct project names
  • Opened / Modified / Closed — distinct dates
  • Status — colour-coded options
  • Tags (collapsible section) — checkboxes for every hashtag used in any history entry

Click Apply to filter the table. An orange "Filter active" banner appears above the table while any filter is enabled. Click Clear in the banner (or Reset in the dialog) to remove all filters.

Search

Click the Search toolbar button to open a search banner below the app bar. Search is full-text and covers: title, project, status, description, related task titles, related design titles, attachment filenames, history entry bodies, and history attachment filenames.

Close the search banner by clicking × or pressing the Search button again.

3.10 Status Distribution Chart

Click the Status Chart (pie chart icon) to open the Status Distribution dialog. It shows a donut chart of tasks grouped by status.

ControlDescription
PeriodLast Day · Last Week · Last Month · Last Year
ProjectAll Projects or a specific project name

The chart legend shows each status, its count, and its percentage. Percentages are labelled on slices ≥ 7%.

3.11 Alarm Calendar

Click the Calendar (calendar icon, green) to open the Alarm Calendar. It displays all tasks that have an active, unfired alarm.

View modes

💡  Double-click any task chip in the calendar to close the calendar and open that task's detail view directly.

5 · Settings Reference

5.1 Installation Path

Access via Settings → Installation Path…. This is the root folder where Memento creates its Memento/ sub-folder. Changing this setting tells Memento where to look for existing databases and where to create new ones.

⚠️  Changing the installation path does not automatically move your existing data. If you have existing tasks or designs, manually move the {OldPath}/Memento/ folder to the new location before clicking Save.

5.2 Command Bar Position

Access via Settings → Command Bar…. Moves the toolbar to one of four positions: Top (default), Bottom, Left, Right. The change takes effect immediately after clicking Apply.

5.3 Language

Access via Settings → Language…. Supported languages:

  • English (default)
  • Italian — all UI text is translated except the words Task, Design, and Tracker, which remain in English for consistency.

Clicking Apply rebuilds the entire UI immediately; no restart is required. The language preference is stored in mem_conf.json.

5.4 Theme

Click the 🌙 / ☀️ icon in the top-right area of the app bar to toggle between Dark and Light mode. The change is instant and persisted immediately. You can also set the initial theme in the first-run wizard.

5.5 Auto-start

Access via Settings → Auto-start…. Controls whether Memento's system-tray process (tray_app.py) is registered to start automatically when Windows boots.

  • Toggle ON — runs python tray_app.py --install in the background, which registers the tray app in the Windows registry startup key.
  • Toggle OFF — runs python tray_app.py --remove in the background, which removes the startup entry.

The preference is saved in mem_conf.json under the key AutoStart (boolean). The same option is available during the first-run wizard (Step 4 of 4).

ℹ️   tray_app.py --install adds an entry to HKCU\Software\Microsoft\Windows\CurrentVersion\Run, so the tray process starts for the current user only, without requiring administrator rights.

6 · System Tray & Alarm Notifications

6.1 Tray Icon & Context Menu

The tray application (tray_app.py) is a lightweight background process that runs in the Windows notification area. It requires pystray and Pillow to be installed.

Start it manually with:

python tray_app.py

Right-clicking the tray icon shows a context menu:

ItemAction
Open Memento (default / double-click)Launches the main window (python main.py)
— separator —
ExitStops the tray process entirely

6.2 Background Alarm Checker

A background daemon thread runs inside the tray process and checks for pending task alarms:

  • An immediate check runs 2 seconds after startup to catch any alarms that fired while the application was closed.
  • Subsequent checks run every 30 seconds.

When a task alarm is due, the tray process fires a Windows toast notification showing:

  • Line 1: Task title
  • Line 2: Project name
  • Icon: Memento application icon

A system sound (MB_ICONEXCLAMATION) also plays. After notification, the alarm is marked as fired in the database so it does not repeat.

6.3 Running at Windows Startup

To make the tray process start automatically with Windows, run:

python tray_app.py --install

This writes a registry entry under:

HKCU\Software\Microsoft\Windows\CurrentVersion\Run\MementoTray

To remove it:

python tray_app.py --remove

ℹ️  The --install / --remove flags only modify the current user's registry hive (HKCU). No administrator privileges are required.

7 · Data & Storage

7.1 Folder Structure

{InstallationPath}/
└── Memento/
    ├── TaskTracker/
    │   ├── db/
    │   │   └── tasks.db
    │   └── attachments/
    │       ├── <task_id>_<filename>
    │       └── ...
    └── DesignTracker/
        ├── db/
        │   └── designs.db
        └── attachments/
            ├── <design_id>_<filename>
            └── ...

Attachment filenames are prefixed with the parent record ID to avoid collisions.

7.2 Configuration File

The configuration is stored in mem_conf.json in the same directory as the Python scripts. All keys and their possible values:

💡  Window position is saved automatically when you move the window, so your preferred placement (including multi-monitor setups) is restored on the next launch.

7.3 Databases

Both trackers use SQLite databases. The databases are created automatically on first use. You can back up your data simply by copying the two .db files and the attachments/ folders.

⚠️  Do not edit the .db files with external SQLite tools while Memento is running, as this may cause data corruption.
