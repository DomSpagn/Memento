Memento

Manuale Utente  ·  Italiano

Memento è un'applicazione desktop per team di ingegneria che devono gestire attività e documenti tecnici in un unico ambiente. Offre due moduli dedicati: il Task Tracker per la gestione delle attività e il Design Tracker per i documenti di progettazione hardware o software (schemi elettrici, layout PCB, block diagram, ecc.).

Tutti i dati sono archiviati localmente in database SQLite all'interno di una cartella a tua scelta. Gli allegati vengono copiati automaticamente in quella cartella. L'applicazione esegue anche un processo in background nella barra delle applicazioni di Windows che invia notifiche toast per gli allarmi dei Task anche quando la finestra principale è chiusa.

Python + Flet SQLite Windows EN / IT

1.2 Procedura guidata di configurazione iniziale

Al primo avvio di Memento (eseguendo python main.py), si apre automaticamente la Procedura guidata perché non esiste ancora alcun file di configurazione. Essa guida l'utente attraverso tre passaggi:

Passo 1 di 4 — Tema

Scegliere tra Modalità scura (impostazione predefinita) e Modalità chiara. L'applicazione applica immediatamente il tema scelto affinché tu possa visualizzarlo in anteprima. La scelta viene salvata nel file di configurazione.

Passo 2 di 4 — Cartella di output

Scegliere la cartella radice in cui Memento archivierà tutti i suoi dati (database, allegati, ecc.). Viene creata automaticamente una sotto-cartella Memento/ nel percorso indicato. Il valore predefinito è ~/Documenti. Fare clic sull'icona cartella per aprire il selettore di directory di sistema.

Passo 3 di 4 — Applicazione di avvio

Selezionare quale modulo mostrare all'avvio di Memento: Task Tracker o Design Tracker. Fare clic su Fine ✓ per completare la procedura guidata; la finestra principale si apre immediatamente.

Passo 4 di 4 — Avvio Automatico

Abilita o disabilita l'avvio automatico del processo in background di Memento all'avvio di Windows. Usare l'interruttore on/off per scegliere la propria preferenza. Se abilitato, il comando python tray_app.py --install viene eseguito immediatamente in background; se disabilitato, non viene eseguita alcuna azione durante la prima configurazione.

ℹ️  Tutte e quattro le preferenze possono essere modificate in seguito tramite il menu Impostazioni.

1.3 Struttura della cartella di output

Al termine della procedura guidata, Memento crea la seguente struttura su disco:

{PercorsoInstallazione}/
└── Memento/
    ├── TaskTracker/
    │   ├── db/
    │   │   └── tasks.db         ← Database SQLite
    │   └── attachments/         ← Allegati dei Task
    └── DesignTracker/
        ├── db/
        │   └── designs.db       ← Database SQLite
        └── attachments/         ← Allegati dei Design

2 · Finestra principale

La finestra principale è composta da una barra dell'applicazione in alto e da un'area di contenuto centrale che commuta tra la vista Task Tracker e la vista Design Tracker.

Sul lato sinistro della barra dell'applicazione è presente un pulsante segmentato con due segmenti:

  • Task Tracker — cursore arancione scuro
  • Design Tracker — cursore blu

Fare clic su un segmento per passare al relativo modulo. L'ultima selezione viene salvata nel file di configurazione (chiave StartWith) e ripristinata al successivo avvio.

2.3 Menu Impostazioni

Fare clic sull'icona ⚙ sul lato destro della barra dell'applicazione per aprire il menu a comparsa Impostazioni. Contiene quattro voci:

  • Percorso d'installazione… — Modifica la cartella radice in cui Memento archivia i dati. Contiene un campo di testo con il percorso corrente e un pulsante Sfoglia…. Pulsanti: Annulla  |  Salva.
  • Posizione barra comandi… — Scegliere dove posizionare la barra degli strumenti (Alto / Basso / Sinistra / Destra). Pulsanti: Annulla  |  Applica.
  • Lingua… — Cambiare la lingua dell'interfaccia tra Inglese e Italiano. Pulsanti: Annulla  |  Applica. L'applicazione della modifica ricostruisce immediatamente l'intera interfaccia.

  • Interruttore tema (icona 🌙 / ☀️) — Commuta tra modalità Scura e Chiara in tempo reale. La preferenza viene salvata immediatamente nel file di configurazione.

    • Informazioni — Mostra la versione dell'applicazione, la data di compilazione e l'autore.
    • Manuale utente — Apre questo manuale.

3 · Task Tracker

Il Task Tracker gestisce le attività di lavoro. Ogni Task possiede un titolo, un progetto, uno stato, un allarme opzionale, una descrizione, elementi correlati, allegati e una cronologia degli aggiornamenti.

3.1 Tabella dei Task

I Task vengono visualizzati in una tabella ordinabile con le seguenti colonne:

Titolo—Link cliccabile (testo arancione) — apre la vista dettaglio
Progetto↕Testo libero
Aperto↕Data e ora ISO, testo grigio
Modificato↕Data e ora ISO, testo grigio
Chiuso↕Popolato quando lo stato è impostato su Chiuso

Valori di stato:

Open In Progress On Hold Closed

Fare clic su una riga per selezionarla (evidenziazione blu) e abilitare i pulsanti Modifica ed Elimina nella barra degli strumenti. Fare clic sulla stessa riga di nuovo per deselezionarla. Fare clic sul link del titolo arancione per aprire il Task senza selezionarlo.

💡  Le intestazioni delle colonne con le frecce ↕ sono cliccabili — fare clic una volta per ordinare in modo crescente, di nuovo per l'ordine decrescente, una terza volta per tornare all'ordine predefinito.

3.2 Creare un nuovo Task

Fare clic sul pulsante Nuovo Task (icona + verde) per aprire la finestra di dialogo Nuovo Task.

Campi obbligatori

  • Titolo — Il nome del Task (messo a fuoco automaticamente all'apertura).
  • Progetto — Il progetto a cui appartiene il Task. Durante la digitazione, un menu a tendina di completamento automatico mostra fino a 6 suggerimenti dai progetti esistenti. Fare clic su un suggerimento per riempire il campo.
  • Stato — Menu a tendina: Open In Progress On Hold Closed. Predefinito: Open.

Il pulsante Salva è disabilitato finché tutti e tre i campi obbligatori non sono compilati.

Campi facoltativi

  • Allarme — Vedere § 3.4 Sezione Allarme.
  • Descrizione — Testo libero multiriga (3–5 righe).
  • Task Correlati — Collega Task esistenti tramite ID.
  • Design Correlati — Collega Design esistenti tramite ID.
  • File — Allega uno o più file.

3.3 Modificare un Task

Aprire un Task facendo clic sul suo link del titolo nella tabella oppure selezionando la sua riga e premendo il pulsante Modifica Task. Si apre la vista dettaglio a pagina intera con una freccia "← Indietro" nella barra dell'applicazione e il numero del Task come titolo di pagina.

Tutti i campi dell'intestazione sono immediatamente modificabili:

CampoTipo
TitoloCampo di testo in linea (larghezza intera, stile sottolineato)
ProgettoCampo di testo in linea + menu a tendina di completamento automatico
StatoMenu a tendina (Open / In Progress / On Hold / Closed)

Ogni modifica viene salvata automaticamente nel database durante la digitazione o la selezione — non è necessario alcun pulsante Salva. Per eliminare definitivamente il Task, fare clic sull'icona 🗑 rossa nell'angolo in alto a destra della sezione intestazione.

3.6 Elementi correlati e allegati

Task Correlati

Nella sezione Task Correlati, digitare il numero ID di un Task nel campo di input e fare clic sul pulsante verde ✓ (o premere Invio) per creare un collegamento bidirezionale tra il Task corrente e quello di riferimento. Ogni Task collegato viene mostrato come: #ID   Titolo Task con un pulsante 🔗 per rimuoverlo.

Messaggi di errore in linea:

  • Inserire un numero valido — input non numerico
  • Già aggiunto — l'ID è già collegato
  • Task non trovato — non esiste nessun Task con quell'ID

Design Correlati

Stesso comportamento dei Task Correlati, ma collega i Design. Inserire un ID Design.

File

Fare clic su Allega file (📎) per aprire un selettore file. Il file scelto viene copiato in {PercorsoInstallazione}/Memento/TaskTracker/attachments/ e rinominato come {nome_originale}_Task_{id}.{ext}, così da essere sempre riconducibile al Task di appartenenza. Ogni allegato è elencato con un nome file cliccabile (apre il file con l'applicazione predefinita del sistema) e un pulsante Elimina (rimuove l'allegato dall'elenco e dal disco). L'allegazione di un file viene salvata immediatamente — non è richiesta alcuna ulteriore azione.

Se il file selezionato è già allegato altrove (in un altro Task, Design o in una qualsiasi voce della cronologia), viene mostrato un avviso con l'elenco di tutti i record che contengono già quel file. In tal caso l'allegato non viene aggiunto.

3.7 Cronologia

La sezione Cronologia in fondo alla vista dettaglio fornisce un registro cronologico degli aggiornamenti. È utile per tracciare note di avanzamento, decisioni e cambiamenti di stato nel tempo.

Aggiunta di un nuovo aggiornamento

  1. Fare clic su + Nuovo aggiornamento per espandere il form di composizione.
  2. Digitare l'aggiornamento nel campo di testo (sfondo bluastro, 2–4 righe).
  3. Aggiungere facoltativamente hashtag: inserire un tag nel campo #tag e fare clic sul pulsante +. I chip del tag appaiono sotto il campo; fare clic su × per rimuoverne uno.
  4. Fare clic facoltativamente sull'icona 📎 per allegare uno o più file all'aggiornamento.
  5. Impostare facoltativamente lo stato dell'aggiornamento tramite il menu a tendina colorato accanto al pulsante di allegato (predefinito: Open).
  6. Fare clic su Salva aggiornamento. I tag vengono aggiunti automaticamente al corpo dell'aggiornamento. È possibile salvare anche con soli allegati, senza inserire testo.
💡  Impostare lo stato di un aggiornamento può aggiornare automaticamente lo stato del Task padre — vedere Calcolo automatico dello stato di seguito.

Aggiornamenti esistenti

Ogni aggiornamento storico è visualizzato come una scheda con:

  • Intestazione: Creato: data relativa + Modificato: data relativa (visualizzata solo se la voce è stata modificata dopo la creazione) + numero aggiornamento + 📎 Allega + 🗑 Elimina + Annulla (✕, visibile durante la modifica) + pulsante Modifica / ✓ Salva alternato.
  • Corpo: il testo dell'aggiornamento (sola lettura per impostazione predefinita). Fare clic su Modifica per attivare la modalità di modifica — la barra di formattazione e la sezione tag appaiono immediatamente. Fare clic su ✓ per salvare. Fare clic su ✕ Annulla per scartare tutte le modifiche e ripristinare il testo originale.
  • File allegati: file collegati a quell'aggiornamento specifico, con gli stessi controlli allega/apri/elimina della sezione File principale.
  • Menu a tendina dello stato: un menu a tendina compatto con testo colorato nell'angolo in basso a destra della scheda. La modifica salva immediatamente il nuovo stato e avvia il calcolo automatico dello stato padre.

Calcolo automatico dello stato

Ogni volta che lo stato di un aggiornamento cambia o un aggiornamento viene eliminato, lo stato del Task padre viene ricalcolato automaticamente con le seguenti regole di priorità:

  1. Almeno un aggiornamento In Progress → Task diventa In Progress.
  2. Almeno un aggiornamento On Hold → Task diventa On Hold.
  3. Tutti gli aggiornamenti Closed → Task diventa Closed.
  4. Tutti gli aggiornamenti Open → Task diventa Open.
  5. Stati misti → lo stato del Task rimane invariato.

3.8 Eliminare un Task

Fare clic sull'icona 🗑 rossa nell'angolo in alto a destra dell'intestazione del Task, oppure selezionare una riga nella tabella e premere il pulsante Elimina Task della barra degli strumenti. Appare una finestra di dialogo di conferma: "Sei sicuro di voler eliminare definitivamente questo Task?"

⚠️  L'eliminazione è permanente. Il record del Task, tutti gli aggiornamenti della cronologia e tutti i file allegati sul disco vengono rimossi. Questa azione non può essere annullata.

3.9 Filtrare e cercare Task

Filtro

Fare clic sul pulsante Filtro della barra degli strumenti per aprire la finestra di dialogo Filtra Task. Usare i menu a tendina per restringere l'elenco:

  • Progetto — nomi di progetto distinti
  • Aperto / Modificato / Chiuso — date distinte
  • Stato — opzioni con codifica colore
  • Tag (sezione espandibile) — caselle di controllo per ogni hashtag usato in qualsiasi riga della cronologia

Fare clic su Applica per filtrare la tabella. Un banner arancione "Filtro attivo" appare sopra la tabella mentre un filtro è abilitato. Fare clic su Cancella nel banner (o Reimposta nella finestra di dialogo) per rimuovere tutti i filtri.

Ricerca

Fare clic sul pulsante Cerca della barra degli strumenti per aprire un banner di ricerca sotto la barra dell'applicazione. La ricerca è full-text e copre: titolo, progetto, stato, descrizione, titoli di Task correlati, titoli di Design correlati, nomi dei file allegati, testi degli aggiornamenti della cronologia e nomi dei file allegati alla cronologia.

Chiudere il banner di ricerca facendo clic su × o premendo di nuovo il pulsante Cerca.

3.10 Grafico distribuzione stati

Fare clic su Grafico stati (icona grafico a torta) per aprire la finestra di dialogo Distribuzione stati. Mostra un grafico a ciambella dei Task raggruppati per stato.

ControlloDescrizione
PeriodoUltimo giorno · Ultima settimana · Ultimo mese · Ultimo anno
ProgettoTutti i progetti oppure un progetto specifico

La legenda del grafico mostra ogni stato, il conteggio e la percentuale. Le percentuali sono etichettate sulle fette ≥ 7%.

3.11 Calendario allarmi

Fare clic sul pulsante Calendario (icona calendario, verde) per aprire il Calendario allarmi. Visualizza tutti i Task con un allarme attivo non ancora attivato.

Modalità di visualizzazione

ModalitàDescrizione
GiornoElenco di tutti gli allarmi del giorno selezionato con i loro orari
SettimanaGriglia a 7 colonne (Lun–Dom); la colonna di oggi è in arancione
MeseGriglia calendario; i giorni con allarmi mostrano punti verdi (massimo 3); oggi evidenziato in arancione

Usare i pulsanti di navigazione ← / → per spostarsi nel tempo, oppure fare clic sull'etichetta del periodo per aprire un selettore anno/mese. Fare clic su Oggi per tornare alla data corrente.

💡  Fare doppio clic su qualsiasi chip Task nel calendario per chiudere il calendario e aprire direttamente la vista dettaglio di quel Task.

4.4 Descrizione, elementi correlati e allegati

Queste sezioni funzionano in modo identico alle loro controparti nel Task Tracker. La vista dettaglio del Design le mostra in quest'ordine: Descrizione → Design Correlati → Task Correlati → File → Cronologia.

Gli allegati vengono copiati in {PercorsoInstallazione}/Memento/DesignTracker/attachments/ e rinominati come {nome_originale}_Design_{id}.{ext}. Vale lo stesso controllo sui file duplicati (vedere § 3.6).

4.5 Cronologia

Il comportamento è identico alla cronologia del Task Tracker (vedere § 3.7), incluso il menu a tendina dello stato per aggiornamento e il calcolo automatico dello stato padre. Scrivere aggiornamenti, aggiungere hashtag e allegare file a ogni voce.

È possibile scrivere aggiornamenti, aggiungere hashtag e allegare file per ogni aggiornamento.

4.6 Eliminare un Design

Fare clic sull'icona 🗑 rossa nell'angolo in alto a destra dell'intestazione del Design, oppure premere il pulsante Elimina Design della barra degli strumenti. Appare una finestra di dialogo di conferma prima dell'eliminazione. Tutti i record del Design, gli aggiornamenti della cronologia e i file allegati sul disco vengono rimossi definitivamente.

4.7 Filtrare e cercare Design

Filtro

La finestra di dialogo Filtro per i Design include tutte le opzioni del Task Tracker più due menu a tendina aggiuntivi:

  • Scheda — nomi di scheda distinti
  • Categoria — Schematic · PCB · Block Diagram · Other
  • Funzione — Connectivity · Logic · HIL · SOM · Other

Ricerca

Ricerca full-text su: nome, scheda, revisione, progetto, categoria, funzione, descrizione, nomi di Design correlati, titoli di Task correlati, nomi dei file allegati, testi della cronologia e nomi dei file allegati alla cronologia.

4.8 Grafico distribuzione stati

Strutturalmente identico al grafico del Task Tracker. Filtra per data di apertura nell'intervallo di tempo selezionato e per progetto.

5 · Riferimento Impostazioni

5.1 Percorso d'installazione

Accedere tramite Impostazioni → Percorso d'installazione…. Questa è la cartella radice in cui Memento crea la sua sotto-cartella Memento/. La modifica di questa impostazione indica a Memento dove cercare i database esistenti e dove crearne di nuovi.

⚠️  La modifica del percorso d'installazione non sposta automaticamente i dati esistenti. Se si hanno Task o Design esistenti, spostare manualmente la cartella {VecchioPercorso}/Memento/ nella nuova posizione prima di fare clic su Salva.

5.2 Posizione barra comandi

Accedere tramite Impostazioni → Posizione barra comandi…. Sposta la barra degli strumenti in una delle quattro posizioni: Alto (predefinito), Basso, Sinistra, Destra. La modifica ha effetto immediatamente dopo aver fatto clic su Applica.

5.3 Lingua

Accedere tramite Impostazioni → Lingua…. Lingue supportate:

  • Inglese (predefinito)
  • Italiano — tutta l'interfaccia utente è tradotta, ad eccezione delle parole Task, Design e Tracker, che rimangono in inglese per coerenza.

Fare clic su Applica ricostruisce immediatamente l'intera interfaccia; non è richiesto alcun riavvio. La preferenza di lingua viene memorizzata in mem_conf.json.

5.4 Tema

Fare clic sull'icona 🌙 / ☀️ nell'area in alto a destra della barra dell'applicazione per passare dalla modalità Scura a quella Chiara e viceversa. Il cambiamento è immediato e viene reso persistente immediatamente. È anche possibile impostare il tema iniziale nella procedura guidata al primo avvio.

5.5 Avvio Automatico

Accedere tramite Impostazioni → Avvio Automatico…. Controlla se il processo in background di Memento (tray_app.py) viene registrato per avviarsi automaticamente all'avvio di Windows.

  • Interruttore ON — esegue python tray_app.py --install in background, registrando il processo nella chiave di avvio del registro di Windows.
  • Interruttore OFF — esegue python tray_app.py --remove in background, rimuovendo la voce di avvio automatico.

La preferenza viene salvata in mem_conf.json con la chiave AutoStart (booleano). La stessa opzione è disponibile durante la procedura guidata al primo avvio (Passo 4 di 4).

ℹ️   tray_app.py --install aggiunge una voce in HKCU\Software\Microsoft\Windows\CurrentVersion\Run, quindi il processo si avvia solo per l'utente corrente, senza richiedere privilegi di amministratore.

6 · Icona di sistema e notifiche allarme

6.1 Icona e menu contestuale

L'applicazione icona di sistema (tray_app.py) è un processo in background leggero che viene eseguito nell'area di notifica di Windows. Richiede l'installazione di pystray e Pillow.

Avviarla manualmente con:

python tray_app.py

Facendo clic destro sull'icona nella barra delle applicazioni si apre un menu contestuale:

VoceAzione
Apri Memento (predefinito / doppio clic)Avvia la finestra principale (python main.py)
— separatore —
EsciTermina completamente il processo di sistema

6.2 Controllo allarmi in background

Un thread daemon in background viene eseguito all'interno del processo di sistema e controlla gli allarmi Task in sospeso:

  • Un controllo immediato viene eseguito 2 secondi dopo l'avvio per rilevare eventuali allarmi scattati mentre l'applicazione era chiusa.
  • I controlli successivi vengono eseguiti ogni 30 secondi.

Quando un allarme Task è in scadenza, il processo invia una notifica toast di Windows che mostra:

  • Riga 1: Titolo del Task
  • Riga 2: Nome del progetto
  • Icona: icona dell'applicazione Memento

Viene anche riprodotto un suono di sistema (MB_ICONEXCLAMATION). Dopo la notifica, l'allarme viene contrassegnato come attivato nel database affinché non si ripeta.

6.3 Avvio automatico con Windows

Per avviare automaticamente il processo di sistema con Windows, eseguire:

python tray_app.py --install

Questo aggiunge una voce nel registro di Windows alla chiave:

HKCU\Software\Microsoft\Windows\CurrentVersion\Run\MementoTray

Per rimuoverla:

python tray_app.py --remove

ℹ️  I flag --install / --remove modificano solo l'hive del registro dell'utente corrente (HKCU). Non sono richiesti privilegi di amministratore.

7.3 Database

Entrambi i moduli usano database SQLite. I database vengono creati automaticamente al primo utilizzo. È possibile eseguire il backup dei dati semplicemente copiando i due file .db e le cartelle attachments/.

⚠️  Non modificare i file .db con strumenti SQLite esterni mentre Memento è in esecuzione, poiché ciò potrebbe causare corruzione dei dati.
